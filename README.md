## 📖 Overview  

**Spiral** is a short narrative-driven point-and-click puzzle game developed during a 3-day Game Jam under the theme **“Nothing Is As It Looks Like.”**

An experienced game jammer reflects on one of her past projects. As she revisits fragmented memories, a lingering sense of unease emerges — something was not quite right… but she just can’t put her finger on it.

Players step into her recollection, uncovering subtle inconsistencies and interacting with curious objects to piece together the truth behind her experience.

---

## 🧠 Core Concept  

- **Genre:** Interactive Fiction / Point-and-Click Puzzle  
- **Theme:** Perception vs Reality  
- **Focus:** Environmental storytelling & subtle narrative clues  

Players progress by:
- Exploring scenes through mouse interaction  
- Clicking on objects to trigger events and dialogue  
- Observing visual inconsistencies and hidden details

## 🛠️ Development Approach  

### 👨‍💻 Programming (Adventure Game Studio - AGS)

The game was developed using Adventure Game Studio (AGS), a classic engine well-suited for point-and-click adventures.

As the programmer, I was responsible for:

- Implementing scene logic and interaction systems  
- Designing clickable hotspots and object behaviors  
- Managing room transitions and narrative triggers  
- Handling animation playback (including GIF-style sequences)  
- Integrating audio cues and background music  
- Structuring game flow using AGS scripting  

#### Key Technical Workflow

1. **Scene Construction**
   - Each scene is built as an AGS *room*
   - Hotspots are mapped to interactive areas

2. **Interaction System**
   - Mouse-based input (click-to-interact)
   - Event-driven scripting for object responses

3. **Asset Integration**
   - Imported 2D assets (PNG, sprite sheets)
   - Animated sequences implemented via:
     - AGS view animations  
     - Frame-based sprite cycling  

4. **Narrative Flow**
   - Dialogue and triggers controlled via scripting
   - Conditional logic for progression  

---

### 🎨 Art & Animation Pipeline  

Our artists created all visual assets, including:

- Background illustrations  
- Character designs  
- Animated sequences  
- UI elements  

#### Integration Process

- Assets exported as PNG sequences or sprite sheets  
- Optimized for AGS compatibility (resolution, transparency)  
- Imported into AGS as sprites and animation views  
- Placed into scenes with correct depth and interaction zones  

Close collaboration ensured:
- Visual consistency  
- Accurate alignment with interaction points  
- Smooth animation playback  

---

### 🎵 Audio & Sound  

Sound design and music were implemented to reinforce the atmosphere:

- Background music loops  
- Ambient sound effects  
- Interaction-triggered audio  

All audio was integrated and triggered through AGS scripting.
